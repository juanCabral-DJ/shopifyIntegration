Integración Shopify para pagos offline, órdenes e inventario

Documento actualizado con base en el desarrollo realizado: backend FastAPI conectado a Shopify Admin API, recepción de webhooks, sincronización de órdenes, confirmación de pagos manuales, inventario y persistencia detallada de productos vendidos por orden.

1. Resumen ejecutivo

La solución implementa un backend para integrar Shopify con procesos internos de confirmación de pagos offline o manuales. El sistema recibe pedidos desde Shopify, identifica métodos como efectivo o transferencia, guarda la información en PostgreSQL y permite confirmar el pago para crear una transacción exitosa en Shopify.

La integración se amplió para incluir inventario y detalle de líneas de pedido. Cada orden ahora puede devolver sus line_items mediante una tabla relacional order_items, lo que facilita consultas, reportes y cruces con producto, variante, SKU e inventario.

6. Webhooks configurados

  • orders/create: crea o actualiza la orden local e identifica pagos offline.
  • orders/updated: mantiene sincronizados estado, datos del cliente y líneas de pedido.
  • inventory_levels/update: actualiza disponibilidad por ubicación.
  • inventory_items/update: actualiza datos del item de inventario.
  • products/update: refresca información de productos/variantes vinculadas al inventario.
Todos los webhooks validan la firma X-Shopify-Hmac-Sha256 antes de procesar el payload.

7. Flujo operativo

  1. Shopify genera o actualiza una orden.
  2. El webhook llega al backend y se valida con HMAC.
  3. La orden se guarda o actualiza en orders.
  4. Los productos vendidos se guardan en order_items.
  5. Si el método de pago es offline, la orden queda en estado interno pending_payment.
  6. Un operador confirma el pago mediante POST /payments/confirm.
  7. El backend crea la transacción en Shopify y marca la orden local como paid.

9. Requisitos de configuración

  • DATABASE_URL: conexión PostgreSQL.
  • SHOPIFY_SHOP: dominio de tienda Shopify.
  • SHOPIFY_ACCESS_TOKEN: token Admin API.
  • SHOPIFY_WEBHOOK_SECRET: secreto para validación HMAC.
  • APP_URL: URL pública para webhooks, normalmente ngrok en desarrollo.
  • Scopes Shopify: read_orders, write_orders, read_customers, read_all_orders, inventario y productos según uso.
